AI · Developer Tools

Apoapsis Harness

A local-first context, research, and verification harness that wraps AI coding agents in a bounded, auditable workflow. A typed agent protocol with no shell or arbitrary command access, deterministic task/spec/plan approval, and risk-based routing that escalates uncertain changes to a human instead of letting a model decide alone.

PythonSQLiteGitOllamaCLI
Read the story

The problem. AI coding agents are useful right up until you give one shell access and it does something you didn't ask for. Most harnesses either lock an agent down until it's useless or trust it with a blast radius no one would sign off on.

The approach. Bound the agent, not the ambition. The coding-agent protocol is typed and shell-free — literal search, bounded reads, diff inspection, and incremental patches, nothing else. Every task moves through a deterministic spec/plan/verify workflow with fail-closed constraint enforcement, so hard requirements can't quietly get dropped.

The build. Risk routing decides, per change, whether it's safe to run local-only, needs a frontier model, or should stop and wait for a human — each path gets its own budget. Everything is logged to an append-only audit trail, and an offline operator UI (task intake, plan approval, human-review queue) runs entirely locally with no CDN or third-party calls.

The outcome. A working harness that treats "the agent decided it was done" as insufficient evidence — the same human-in-the-loop discipline behind AutoApply's own review gate, generalized into reusable infrastructure for any AI coding agent.

AI · Audio · In classroom use

prof-ai

A voice-interactive lecture assistant that transcribes class live, spots gaps in the explanation, and answers questions out loud using the professor's own course materials. It runs entirely on local hardware, so there are no per-query costs and no student data ever leaves the room. A Baylor professor is using it in class right now, and the university is looking at wider adoption.

PythonOllamaChromaDBfaster-whisperPiper TTSRAG
Read the story

The problem. A cloud AI assistant in a live classroom means streaming students' voices and a professor's unpublished materials to someone else's servers. For most instructors that's a non-starter, and rightly so.

The approach. Everything runs locally. Audio is transcribed on-device with faster-whisper, the professor's own slides and readings become the knowledge base in ChromaDB, a local model answers through Ollama, and Piper speaks the reply back. Nothing leaves the machine, so there was no budget or procurement barrier to adoption.

The build. I designed it as a lecture / question / processing state machine and iterated directly with the professor using it, adding gap detection and a multi-turn conversation mode along the way.

The outcome. It's in active classroom use, and the university is weighing wider adoption.

AI · Image editing

Kitchen Remix

Take a photo of a real kitchen and preview remodel options (cabinets, countertops, flooring, paint) as photorealistic edits, before spending a dollar. Runs FLUX.1 Kontext locally on an RTX 4090, so photos never leave the machine. It sorts every edit into "scope tiers" that say plainly where the model is reliable and where it's guessing.

FLUX.1 KontextReactFastAPICUDA
Read the story

The problem. Most people remodeling a kitchen can't picture the result, and designer renders cost time and money. I built this for my parents, who wanted to compare a few directions before committing.

The hard part. AI image tools will cheerfully generate anything, including changes the model has no basis for. A confident render of a wall that isn't there invites a real decision based on a guess. So the design question wasn't "what can the model do?" but "what can it do reliably, and how do we label the rest?"

The answer. Every edit is sorted into scope tiers. Core edits (finish swaps, object removal) need no information the photo doesn't already contain, so they're treated as dependable. Experimental edits (moving objects, opening walls) invent geometry, so they're labeled "not dimensionally accurate." A reused random seed across an A/B set keeps the comparison fair by changing only the finish, not the scene.

Why it's local. FLUX.1 Kontext runs on a home RTX 4090, loaded once and kept warm in VRAM, so photos never leave the machine and inference stays fast.

Research Proposal · Presented to the National Society in Neuroscience

Training-Induced Occipital Recruitment in Late-Blind Adults

A full fMRI study proposal testing whether eight weeks of multisensory navigation training can re-engage the visual cortex in adults who lost sight after childhood. It includes hypotheses, a GPower-based power analysis, a sensory-substitution apparatus design, an MRI acquisition and preprocessing protocol, and a full statistical plan, all built on the cross-modal plasticity literature.

Capstone · NSC 4330, Technology & the Nervous System

From Affect to Merge: How Technology Is Rewiring the Brain

A semester-long synthesis of about 20 papers, traced along one arc of technology moving from the outside in: affecting behavior, exploiting reward circuits, reading the brain, stimulating it, and finally merging with it (brain-spine interfaces, hippocampal prosthetics, the MICrONS connectome). It closes on the open questions around neural-data rights and the gap between predicting behavior and consciousness.

Journal Club · Neural Stimulation

TMS & Visual Working Memory

Analysis of a cerebellar TMS study (Viñas-Guasch et al., 2023) showing that disrupting the left inferior cerebellum selectively impairs visual working memory but leaves verbal memory intact. I broke down the fMRI-guided targeting, the double-dissociation design, the statistics, and where the study's small sample limits its claims.
